# Pull Request Checklist

Use this checklist to keep PRs small, reviewable, and easy to merge.

## Before opening a PR

- [ ] The change is focused (one topic).
- [ ] I described *what* changed and *why*.
- [ ] Screenshots are included if UI changed.

## Quality

- [ ] I updated or added docs if needed.
- [ ] I added/updated tests if the change affects behavior.
- [ ] I checked for obvious typos and formatting issues.
