# Contributing via GitHub UI (No Terminal)

This repo accepts small improvements via the GitHub web interface.

## Quick steps
1. Fork the repository
2. Create a new branch in your fork
3. Edit or add a file
4. Commit changes to your branch
5. Open a Pull Request

## Before you open a PR
- Keep the change small and focused
- Update docs only (no code changes) if you are unsure
- Add screenshots only when helpful
- Use clear, short titles

## What is a good first PR?
- Fix typos / unclear wording
- Add missing links
- Improve README/Docs structure
- Add small FAQ entries
